Understanding FigML's XML Structure

FigML converts your Figma designs into a clean, structured XML format that preserves the essential hierarchy and styling information while eliminating unnecessary complexity.

Basic Structureâ€‹

The XML output from FigML follows a hierarchical structure that mirrors your Figma design:

<Component name="Card" type="FRAME">
  <Frame name="Header" width="320" height="48">
    <Text name="Title" fontSize="16" fontWeight="600">Card Title</Text>
  </Frame>
  <Frame name="Content" width="320" height="120">
    <Text name="Description">Card description goes here</Text>
    <Button name="Action" width="80" height="32">Click Me</Button>
  </Frame>
</Component>

Element Typesâ€‹

FigML preserves the original Figma element types in the XML output:

Figma ElementXML Tag
Frame<Frame>
Group<Group>
Text<Text>
Component<Component>
Instance<Instance>
Rectangle<Rectangle>
Vector<Vector>
Line<Line>

Attributesâ€‹

Each element includes attributes that represent its properties:

Common Attributesâ€‹

These attributes appear on most elements:

  • name: The name of the element in Figma
  • width: Width in pixels
  • height: Height in pixels
  • x: X position relative to parent
  • y: Y position relative to parent
  • visible: Whether the element is visible (true/false)

Style Attributesâ€‹

Style attributes are included when relevant:

  • fill: Fill color in hex format (#RRGGBB)
  • opacity: Opacity value (0-1)
  • stroke: Stroke color in hex format
  • strokeWeight: Stroke width in pixels
  • cornerRadius: Corner radius for rounded elements

Text-Specific Attributesâ€‹

Text elements include additional attributes:

  • fontSize: Font size in pixels
  • fontFamily: Font family name
  • fontWeight: Font weight (400, 500, 700, etc.)
  • textAlignHorizontal: Text alignment (LEFT, CENTER, RIGHT)
  • textAlignVertical: Vertical alignment (TOP, CENTER, BOTTOM)
  • lineHeight: Line height in pixels or percentage

Layout Informationâ€‹

FigML preserves layout constraints and auto-layout properties:

<Frame name="Container" layout="VERTICAL" spacing="16" padding="24">
  <Text name="Title" layoutAlign="STRETCH">Title Text</Text>
  <Frame name="Content" layoutAlign="STRETCH">
    <!-- Content elements -->
  </Frame>
</Frame>

Layout attributes include:

  • layout: Layout type (NONE, HORIZONTAL, VERTICAL)
  • layoutAlign: How the element aligns in its parent (STRETCH, CENTER, etc.)
  • spacing: Space between children in auto-layout
  • padding: Padding inside the container (can be specified as top, right, bottom, left)

Component Instancesâ€‹

Component instances reference their main component:

<Instance name="Button/Primary" componentId="button-primary">
  <Text name="Label">Button Text</Text>
</Instance>

Benefits for Developmentâ€‹

This XML structure is optimized for:

  1. Readability: Clean, human-readable format
  2. AI Processing: Structured format that AI tools can easily understand
  3. Code Generation: Contains the necessary information for generating UI code
  4. Simplicity: Removes Figma's complex JSON structure while preserving essential information

Example: Card Componentâ€‹

Here's a complete example of a card component in FigML's XML format:

<Component name="Card" type="FRAME" width="320" height="240" fill="#FFFFFF" cornerRadius="8" stroke="#EEEEEE" strokeWeight="1">
  <Frame name="CardHeader" width="320" height="48" padding="16">
    <Text name="Title" fontSize="16" fontWeight="600" textAlignVertical="CENTER">Card Title</Text>
  </Frame>
  <Frame name="CardContent" width="320" height="140" padding="16">
    <Text name="Description" fontSize="14" lineHeight="20" opacity="0.8">
      This is a sample card description that demonstrates the XML structure of a typical component.
    </Text>
    <Frame name="Actions" width="320" height="40" layout="HORIZONTAL" spacing="8">
      <Instance name="Button/Secondary" componentId="button-secondary" width="80" height="32">
        <Text name="Label">Cancel</Text>
      </Instance>
      <Instance name="Button/Primary" componentId="button-primary" width="80" height="32">
        <Text name="Label">Submit</Text>
      </Instance>
    </Frame>
  </Frame>
</Component>

This structured format makes it easy to understand the component hierarchy and translate it into code for any framework.
